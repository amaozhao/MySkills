# 通用文章写作助手

## 概述

本助手提供从选题到复盘全流程的文章写作支持，包含五个核心模块，可根据不同平台需求灵活配置使用。

## 使用场景

- 不知道写什么，需要选题建议
- 确定选题后，需要大纲模板
- 写完文章，需要标题优化
- 文章完成，准备发布
- 发布后，需要数据分析

## 核心模块

### 模块一：选题助手

当你不确定写什么主题时使用。选题助手会引导你完成5问决策流程，评估选题价值，给出具体选题建议。

### 模块二：大纲助手

当你确定选题后使用。大纲助手会根据文章类型生成对应的大纲模板，帮助你快速搭建文章结构。

### 模块三：标题助手

当你准备写文章或文章完成后使用。标题助手会生成多个备选标题，并给出评估建议。

### 模块四：发布助手

当你写完文章准备发布时使用。发布助手会根据目标平台和受众特性给出发布时间建议和发布前检查清单。

### 模块五：复盘助手

当你发布文章一段时间后使用。复盘助手会分析数据、总结经验、给出改进建议。

## 使用方式

### 输入参数

```json
{
  "modules": ["选题助手", "大纲助手", "标题助手"],
  "article_type": "热点解读",
  "platform": "公众号",
  "has_material": false
}
```

### 参数说明

| 参数 | 必填 | 可选值 | 说明 |
|------|------|--------|------|
| modules | 是 | 选题助手、大纲助手、标题助手、发布助手、复盘助手 | 可多选，用逗号分隔 |
| article_type | 否 | 工具对比、情绪共鸣、热点解读、日常感悟 | 文章类型 |
| platform | 否 | 公众号、知乎、小红书、博客等 | 目标发布平台 |
| has_material | 否 | true、false | 是否有现成素材 |

### 快捷指令

| 指令 | 含义 |
|------|------|
| 帮我选题 | 只调用选题助手 |
| 帮我起标题 | 只调用标题助手 |
| 准备发布 | 只调用发布助手 |
| 分析数据 | 只调用复盘助手 |

## 文章类型与写作模板

### 类型一：工具对比文

**适用场景**：比较两个或多个工具、产品的优劣。

**读者价值**：帮助读者做出选择决策。

**大纲模板**：

```
【标题】使用XX和XX后，我发现了X个关键差异

【开头（150字以内）】
- 场景代入：什么场景下使用的
- 发现：有什么问题或重要发现
- 预告：本文会告诉你什么

【正文结构】
一、先说结论（300字）
- 两个工具的定位和特点
- 我的推荐（针对不同需求）

二、核心对比（800字）
- 功能对比
- 性能对比
- 性价比对比
- 每个对比点用实际案例说明

三、适用场景（500字）
- 什么情况用A更好
- 什么情况用B更好
- 什么情况可以都用

四、最终建议（300字）
- 明确推荐
- 给出使用建议

【结尾】
- 总结核心观点
- 引导互动讨论
```

**字数参考**：2000-3000字

### 类型二：情绪共鸣文

**适用场景**：讲述个人经历，引发读者情感共鸣。

**读者价值**：获得情感支持、行动启发。

**大纲模板**：

```
【标题】关于XX的那段经历，我有些话想说

【开头（200字以内）】
- 场景：具体的时间、地点、情境
- 情绪：那一刻的真实感受
- 转折：我决定如何面对

【正文结构】
一、那个瞬间（400字）
- 详细描述发生了什么
- 细节越多越好（增强代入感）
- 真实呈现，不刻意渲染

二、为什么难（500字）
- 不是简单的困难，是深层的挑战
- 触及真正的痛点
- 引发读者共情

三、我做了什么（600字）
- 经历了哪些挣扎和尝试
- 如何一步步走出来的
- 关键转折点和突破

四、给同样处境的你（500字）
- 不是空洞的鸡汤，是具体的建议
- 可操作性强
- 真诚不夸大

【结尾】
- 一句话总结核心感悟
- 引导互动交流
```

**字数参考**：1800-2500字

### 类型三：热点解读文

**适用场景**：解读当前热点事件，提供独特视角。

**读者价值**：了解事件背后的逻辑、获得独特观点。

**大纲模板**：

```
【标题】关于XX，我来分享一些不同的看法

【开头（150字以内）】
- 简单交代发生了什么
- 你的独特立场或角度

【正文结构】
一、发生了什么（300字）
- 客观描述事件经过
- 关键时间节点

二、为什么引发关注（400字）
- 分析传播原因
- 不同角度看问题

三、深度解读（800字）
- 你的独特分析角度
- 从多个层面剖析
- 不是简单复述，是深度思考

四、对目标读者的意义（500字）
- 利好哪些人
- 需要注意什么
- 有什么启示

五、我的判断和建议（300字）
- 趋势预测
- 行动建议

【结尾】
- 核心观点总结
- 引导讨论交流
```

**字数参考**：2000-3000字

### 类型四：日常感悟文

**适用场景**：分享日常生活感悟、心得。

**读者价值**：获得启发、情感共鸣。

**大纲模板**：

```
【标题】关于XX，我的一点思考

【开头（150字以内）】
- 引入话题
- 抛出核心观点

【正文结构】
一、背景交代（300字）
- 发生了什么
- 我的处境和情境

二、核心观点（600字）
- 我的深入思考
- 我的实践和体验
- 具体的收获和感悟

三、经验总结（400字）
- 我学到了什么
- 对你可能有何启发

【结尾】
- 升华主题
- 引导互动交流
```

**字数参考**：1500-2000字

## 标题优化方法

### 标题核心要素

| 要素 | 说明 | 示例 |
|------|------|------|
| 情绪词 | 引发情感反应 | 震惊、终于、竟然、可惜 |
| 数字 | 具体明确 | 3个技巧、7天、100% |
| 悬念 | 引发好奇心 | 背后的秘密、没想到 |
| 价值承诺 | 明确告诉读者好处 | 学会、避坑、省钱 |

### 标题生成技巧

1. **数字+痛点+解决方案**：如"3个方法解决XX问题"
2. **对比+冲突**：如"XX和XX的差距有多大"
3. **悬念+独家**：如"XX背后的真相"
4. **身份标签+共鸣**：如"XX人一定要看"

### 标题评估标准

- **相关性**：是否准确反映文章内容
- **吸引力**：是否能引起目标读者注意
- **可读性**：是否容易理解和记忆
- **差异化**：是否与同类文章有区分度

## 发布策略

### 发布前检查清单

**内容检查**：
- [ ] 开头有吸引力（钩子）
- [ ] 有至少1个真实案例或故事
- [ ] 有实质性的内容或观点
- [ ] 结尾有总结和引导

**标题检查**：
- [ ] 标题能引起目标读者兴趣
- [ ] 标题清晰表达文章价值
- [ ] 标题无歧义

**排版检查**：
- [ ] 有合适的配图
- [ ] 段落分隔清晰
- [ ] 重点内容有标识

**平台适配检查**：
- [ ] 符合目标平台的内容规范
- [ ] 标签和分类选择合适
- [ ] 发布时间选择合适

### 发布时机选择

根据目标平台的受众活跃时间选择合适的发布时间。可参考以下原则：

- **工作日早晨**：通勤场景，用户有碎片时间
- **午休时间**：用户相对放松
- **晚间黄金时段**：用户注意力集中，适合深度内容
- **周末**：用户有更充裕的时间

具体最佳时间需要根据自己账号的数据不断优化。

## 数据分析

### 核心数据指标

| 指标 | 含义 | 参考价值 |
|------|------|----------|
| 阅读量/浏览量 | 内容曝光度 | 反映标题吸引力+分发效果 |
| 互动率 | 用户参与程度 | 反映内容质量 |
| 分享率 | 传播能力 | 反映内容价值 |
| 完读率 | 阅读完成度 | 反映内容吸引力 |
| 评论数 | 用户反馈 | 反映话题讨论度 |

### 数据复盘维度

| 维度 | 分析要点 |
|------|----------|
| 选题 | 是否击中目标读者需求 |
| 标题 | 是否有足够的吸引力 |
| 开头 | 是否成功抓住读者 |
| 内容 | 是否有实际价值 |
| 结尾 | 是否有有效引导 |
| 时机 | 是否选对发布时间 |
| 配图 | 是否有视觉吸引力 |

### 改进方向

| 问题表现 | 改进策略 |
|----------|----------|
| 阅读量低 | 优化标题、改进选题 |
| 互动率低 | 增加干货、强化观点输出 |
| 分享率低 | 增强情感共鸣、结尾引导转发 |
| 完读率低 | 优化开头、增强内容吸引力 |
| 评论少 | 结尾提开放性问题 |

## 时间管理

### 单篇文章时间分配

| 阶段 | 时间 | 建议 |
|------|------|------|
| 选题确认 | 30分钟 | 使用5问决策法 |
| 资料收集 | 1-2小时 | 根据选题调整 |
| 大纲设计 | 30分钟 | 先搭框架再填内容 |
| 初稿写作 | 2-4小时 | 专注完成初稿 |
| 修改优化 | 1小时 | 优化表达和逻辑 |
| 发布准备 | 30分钟 | 配图和排版 |
| **总计** | **6-9小时** | 根据个人情况调整 |

### 写作效率建议

1. **固定写作时间**：找到自己的高效写作时段
2. **批量处理**：同类型工作集中处理
3. **番茄工作法**：25分钟专注+5分钟休息
4. **先完成后完美**：初稿不求完美，先完成再优化

## 常见问题

### 问题一：不知道写什么怎么办？

**答案**：从生活中找素材。从你的真实经历、遇到的问题、学到的东西入手。真实的故事最有共鸣。

### 问题二：写不出来怎么办？

**答案**：先写大纲，大纲是骨架，先把骨架搭起来，再填肉。有大纲在手，写作会顺畅很多。

### 问题三：阅读量一直很低怎么办？

**答案**：检查标题是否有吸引力，检查开头是否抓住人，检查内容是否有干货。也可以研究同类高阅读量文章的做法。

### 问题四：如何保持持续输出？

**答案**：建立选题库，随时记录灵感。制定写作计划，固定写作时间。追求进步，不追求完美。

### 问题五：数据焦虑怎么办？

**答案**：发完一段时间后再看数据，把精力放在下一篇文章上。数据是参考，不是唯一标准。

## 注意事项

本助手能做的：
- 选题分析和建议
- 大纲模板生成
- 标题建议
- 发布策略建议
- 数据分析和复盘

本助手不能做的：
- 直接生成完整文章（需要你自己写）
- 配图生成（只能给建议）
- 代替你做决策（只给建议）
- 保证阅读量（取决于内容和执行）

## 平台配置参考

### 微信公众号

- **发布最佳时间**：早7-8点、午12-13点、晚20-22点
- **配图建议**：16:9封面图，正文配图3-5张
- **内容特点**：允许较长篇幅，注重深度

### 知乎

- **发布最佳时间**：晚间黄金时段
- **配图建议**：竖图为主，支持live图片
- **内容特点**：注重逻辑和专业性

### 小红书

- **发布最佳时间**：午12-13点、晚20-22点
- **配图建议**：竖图4:3为主，强调视觉效果
- **内容特点**：短平快，注重实用性和颜值

### 博客

- **发布最佳时间**：根据博客定位和受众确定
- **配图建议**：根据文章主题选择
- **内容特点**：可长可短，形式灵活
