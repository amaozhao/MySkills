# 写作助手

## 概述

帮助你顺利完成写作，提供开头、正文、结尾的具体写法指导。

## 工作流程

```
大纲确认 → 开头写作 → 正文写作 → 结尾写作 → 自查完成
```

## E-E-A-T 信任要素

借鉴 seo-content-writer，添加信任度要素：

| 要素 | 说明 | 示例 |
|------|------|------|
| **E**xperience | 亲身经历 | "我使用 Trae 一周后的真实体验..." |
| **E**xpertise | 专业背景 | "作为 5 年经验的开发者..." |
| **A**uthoritativeness | 权威引用 | "根据 Stack Overflow 2025 调查..." |
| **T**rustworthiness | 可信来源 | "实测数据来源：本地测试环境..." |

## 开头写法

开头决定读者是否继续阅读。前 100 字必须有足够的吸引力。

### 方法一：故事引入法

**原理**：用一个小故事快速建立场景，让读者产生代入感。

**模板**：
```
【时间】+【地点】+【动作】+【转折】
```

**示例**：
> 去年这个时候，我在北京挤地铁的时候，突然收到一条消息...

**注意事项**：
- 故事要短，控制在 100 字以内
- 要有具体细节（时间、地点、动作）
- 结尾要有悬念或转折

### 方法二：问题切入法

**原理**：提出一个读者关心的问题，引发共鸣和好奇。

**模板**：
```
【痛点描述】+【引发思考】
```

**示例**：
> 有没有发现，身边越来越多的人开始用 AI 工具了？

### 方法三：数据震撼法

**原理**：用一个令人惊讶的数据开场，建立权威感。

**模板**：
```
【数据】+【数据解读】+【核心观点】
```

**示例**：
> 调查显示，80% 的职场人每天要花 2 小时在重复性工作上...

### 方法四：场景还原法

**原理**：描绘一个读者熟悉的生活场景，引发共鸣。

**模板**：
```
【场景描述】+【情感共鸣】+【引出主题】
```

## 正文展开技巧

### 核心法则：观点 + 案例 + 总结

```
【观点陈述】50字以内
  ↓
【案例说明】100-200字
  ↓
【一句话总结】20-30字
```

**示例**：
> 很多人在选择工具时，容易犯的第一个错误就是只看价格。（观点）
> 我之前也这样，觉得免费的就是最好的...
> 直到后来我发现，免费的往往是最贵的。（总结）

### 段落衔接技巧

1. **使用过渡句**
   ```
   说完A观点 → "但是，仅仅A还不够..."
   说完B观点 → "更重要的是..."
   ```

2. **控制段落长度**
   - 重点论述：200-300字
   - 过渡段落：50-100字
   - 观点总结：30-50字

3. **段落首句要精炼** - 每段的第一句话要能概括整段的核心意思

### 案例写作技巧

1. **细节越具体越好**
   - ❌ "那天我很紧张"
   - ✅ "那天我的手心全是汗，说话时声音都在抖"

2. **用对话增加真实感** - 加入人物对话，让案例更生动

3. **案例要指向观点** - 每一个案例都要说明它支持哪个观点

## 结尾写法

结尾要完成两个任务：总结全文 + 引导行动。

### 方法一：观点升华法

**结构**：总结核心观点 + 金句升华

**示例**：
> 回头看这篇文章，我想说的核心就是...
> 工具只是工具，真正决定效率的，是使用工具的人。

### 方法二：提问引导法

**结构**：提出开放性问题 + 邀请互动

**示例**：
> 你有没有类似的经历？欢迎在评论区分享你的故事。

### 方法三：行动号召法

**结构**：总结收获 + 鼓励行动

**示例**：
> 如果这篇文章对你有帮助，欢迎「点赞」+「在看」，我们下期再见。

## 写作速度提升技巧

### 1. 番茄写作法

- 25 分钟专注写作 + 5 分钟休息
- 一个番茄完成 500-800 字初稿
- 4 个番茄完成一篇 2000-3000 字文章

### 2. 自由写作法

- 不管写得好不好，先写出来
- 想到什么写什么，不要停下来
- 初稿不追求完美，先完成再修改

### 3. 模块化写作法

- 先写最简单的部分
- 难的留到最后
- 最后统一修改

## 常见写作问题解决

| 问题 | 原因 | 解决方法 |
|------|------|----------|
| 不知道第一句写什么 | 开头压力太大 | 先写第二段，最后补开头 |
| 写着写着跑题了 | 缺乏锚点 | 随时对照大纲，检查是否偏离 |
| 不知道怎么过渡 | 逻辑不清 | 用"连接词+观点"的结构 |
| 写得太慢 | 追求完美 | 先完成初稿，再修改润色 |
| 感觉写得很干 | 缺乏案例 | 暂停写作，先找 2-3 个案例 |

## 写作状态调整

### 进入状态的方法

1. **仪式感**：固定时间、固定位置
2. **热身**：写 100 字随便什么内容
3. **目标明确**：告诉自己"今天只写 1000 字"

### 卡住时的处理

1. **换一段写**：跳过卡住的地方
2. **大声朗读**：听听是否通顺
3. **休息一下**：散步 5 分钟再回来
4. **降低标准**：允许自己写垃圾
