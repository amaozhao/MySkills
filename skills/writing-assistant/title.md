# 标题助手

## 概述

帮助你生成吸引人的文章标题，并提供评估建议。

## 工作流程

```
写作完成 → 生成标题备选（3-5个）→ 标题评估 → 选择确认 → 进入发布
```

## 输出规范

借鉴 seo-content-writer，每次输出：

```
【标题输出规范】

主标题备选（3个）：
1. [标题1]
2. [标题2]
3. [标题3]

副标题备选（2个）：
1. [副标题1]
2. [副标题2]

推荐标题：[编号]
推荐理由：[为什么这个最好]

微信公众号格式：
- 主标题：16字以内
- 副标题：36字以内

知乎格式：
- 标题：25字以内
- 问题式更佳

小红书格式：
- 标题：20字以内
- 可用 emoji
```

---

## 标题核心要素

| 要素 | 说明 | 示例 |
|------|------|------|
| 情绪词 | 引发情感反应 | 震惊、终于、竟然、可惜 |
| 数字 | 具体明确 | 3个技巧、7天、100% |
| 悬念 | 引发好奇心 | 背后的秘密、没想到 |
| 价值承诺 | 明确告诉读者好处 | 学会、避坑、省钱 |

## 标题生成技巧

1. **数字+痛点+解决方案**：如"3个方法解决XX问题"
2. **对比+冲突**：如"XX和XX的差距有多大"
3. **悬念+独家**：如"XX背后的真相"
4. **身份标签+共鸣**：如"XX人一定要看"

## 标题评估标准

| 标准 | 说明 |
|------|------|
| 相关性 | 是否准确反映文章内容 |
| 吸引力 | 是否能引起目标读者注意 |
| 可读性 | 是否容易理解和记忆 |
| 差异化 | 是否与同类文章有区分度 |

## 标题备选示例

```
【标题备选】
1. 用了Cursor和Trae后，我发现了5个关键差异
2. Cursor vs Trae：谁才是AI编程之王的真相
3. 从VS Code切换到Cursor一周后，我后悔了
4. 三个插件，让我的编码效率翻倍

【推荐】标题1
理由：数字明确、冲突感强、价值承诺清晰
```

## 按平台调整

| 平台 | 标题风格 |
|------|---------|
| 公众号 | 长标题、情绪词多、价值明确 |
| 知乎 | 问题式、对比式 |
| 小红书 | 短标题、emoji、悬念 |
| 博客 | 关键词明确、SEO友好 |
