# 微信公众号写作助手

## 概述

本助手提供从选题到发布全流程的微信公众号写作支持，包含五个独立模块，可根据需求自由组合使用。

## 使用场景

- 不知道写什么，需要选题建议
- 确定选题后，需要大纲模板
- 写完文章，需要标题优化
- 文章完成，准备发布
- 发布后，需要数据分析

## 模块结构

### 模块一：选题助手

当你不确定写什么主题时使用。选题助手会自动搜索当前热门话题，评估选题价值，生成多个备选选题供用户选择。

**工作流程**：
1. 搜索当前热门话题（技术圈、社交媒体、行业动态）
2. 分析热度、时效性、独特角度
3. 生成 3-5 个备选选题
4. 用户选择后，进入选题决策 5 问流程

### 模块二：大纲助手

当你确定选题后使用。大纲助手会根据文章类型（工具对比、情绪共鸣、热点解读）生成对应的大纲模板。

### 模块三：写作助手

当你开始写初稿或写不下去时使用。写作助手会提供开头、正文、结尾的具体写法指导，帮助你顺利完成写作。

### 模块四：标题助手

当你准备写文章时使用。标题助手会生成多个备选标题，并给出评估建议。

### 模块四：发布助手

当你写完文章准备发布时使用。发布助手会给出发布时间建议、配图清单、发布前检查清单。

### 模块五：复盘助手

当你发布文章48小时后使用。复盘助手会分析数据、总结经验、给出改进建议。

## 使用方式

### 输入参数

```json
{
  "modules": ["选题助手"],
  "focus_area": "AI工具",
  "time_budget": "2-3小时",
  "has_material": false,
  "deadline": "本周"
}
```

### 参数说明

| 参数 | 必填 | 可选值 | 说明 |
|------|------|--------|------|
| modules | 是 | 选题助手、大纲助手、写作助手、标题助手、发布助手、复盘助手 | 可多选，用逗号分隔 |
| focus_area | 否 | AI工具、前端开发、后端开发、职场、个人成长 | 关注领域（不指定则搜索全领域） |
| time_budget | 否 | 2-3小时、3-4小时、5小时+ | 时间预算 |
| has_material | 否 | true、false | 是否有现成素材 |
| deadline | 否 | 今天、明天、本周 | 发布截止时间 |

### 快捷指令

| 指令 | 含义 |
|------|------|
| 写热点文 | 自动调用选题助手＋大纲助手＋标题助手 |
| 帮我起标题 | 只调用标题助手 |
| 准备发布 | 只调用发布助手 |
| 分析数据 | 只调用复盘助手 |
| 完整流程 | 调用全部五个模块 |

## 写作助手

### 什么是写作助手

写作助手是连接大纲和标题的中间环节。当你有了大纲却不知如何下笔，或者写到一半卡住时，写作助手提供具体可操作的写作方法。

### 适用场景

- 大纲完成后，不知道怎么写开头
- 写了一段，后面不知道怎么接
- 写完初稿，需要系统化检查

### 开头写法详解

开头决定读者是否继续阅读。前100字必须有足够的吸引力。

#### 方法一：故事引入法

**原理**：用一个小故事快速建立场景，让读者产生代入感。

**模板**：
```
【时间】+【地点】+【动作】+【转折】
```

**示例**：
> 去年这个时候，我在北京挤地铁的时候，突然收到一条消息...

**注意事项**：
- 故事要短，控制在100字以内
- 要有具体细节（时间、地点、动作）
- 结尾要有悬念或转折，引导继续阅读

#### 方法二：问题切入法

**原理**：提出一个读者关心的问题，引发共鸣和好奇。

**模板**：
```
【痛点描述】+【引发思考】
```

**示例**：
> 有没有发现，身边越来越多的人开始用AI工具了？

**注意事项**：
- 问题要击中目标读者的痛点
- 问题后要有简短的思考引导
- 避免明知故问式的明知故问

#### 方法三：数据震撼法

**原理**：用一个令人惊讶的数据开场，建立权威感。

**模板**：
```
【数据】+【数据解读】+【核心观点】
```

**示例**：
> 调查显示，80%的职场人每天要花2小时在重复性工作上...

**注意事项**：
- 数据来源要可靠
- 数据解读要有洞察
- 数据要和后文内容相关

#### 方法四：场景还原法

**原理**：描绘一个读者熟悉的生活场景，引发共鸣。

**模板**：
```
【场景描述】+【情感共鸣】+【引出主题】
```

**示例**：
> 周一早上，闹钟响了第3遍，你终于起床...
> 打开电脑，发现又是满满的工作邮件...

**注意事项**：
- 场景要足够具体和真实
- 要能唤起读者的情感反应
- 场景要和文章主题相关

### 正文展开技巧

#### 核心法则：观点 + 案例 + 总结

每一个小节都应该遵循这个结构：

```
【观点陈述】50字以内
  ↓
【案例说明】100-200字
  ↓
【一句话总结】20-30字
```

**示例**：
> 很多人在选择工具时，容易犯的第一个错误就是只看价格。（观点）
> 我之前也这样，觉得免费的就是最好的...
> 直到后来我发现，免费的往往是最贵的。（总结）

#### 段落衔接技巧

1. **使用过渡句**
   ```
   说完A观点 → "但是，仅仅A还不够..."
   说完B观点 → "更重要的是..."
   ```

2. **控制段落长度**
   - 重点论述：200-300字
   - 过渡段落：50-100字
   - 观点总结：30-50字

3. **段落首句要精炼**
   每段的第一句话要能概括整段的核心意思。

#### 案例写作技巧

1. **细节越具体越好**
   - ❌ "那天我很紧张"
   - ✅ "那天我的手心全是汗，说话时声音都在抖"

2. **用对话增加真实感**
   - 加入人物对话，让案例更生动

3. **案例要指向观点**
   - 每一个案例都要说明它支持哪个观点
   - 案例讲完后要有分析，不能只讲故事

### 结尾写法详解

结尾要完成两个任务：总结全文 + 引导行动。

#### 方法一：观点升华法

**结构**：总结核心观点 + 金句升华

**示例**：
> 回头看这篇文章，我想说的核心就是...
> 工具只是工具，真正决定效率的，是使用工具的人。

#### 方法二：提问引导法

**结构**：提出开放性问题 + 邀请互动

**示例**：
> 你有没有类似的经历？欢迎在评论区分享你的故事。

#### 方法三：行动号召法

**结构**：总结收获 + 鼓励行动

**示例**：
> 如果这篇文章对你有帮助，欢迎「点赞」+「在看」，我们下期再见。

### 写作速度提升技巧

#### 1. 番茄写作法

- 25分钟专注写作 + 5分钟休息
- 一个番茄完成 500-800 字初稿
- 4个番茄完成一篇 2000-3000 字文章

#### 2. 自由写作法

- 不管写得好不好，先写出来
- 想到什么写什么，不要停下来
- 初稿不追求完美，先完成再修改

#### 3. 模块化写作法

- 先写最简单的部分
- 难的留到最后
- 最后统一修改

### 常见写作问题解决

| 问题 | 原因 | 解决方法 |
|------|------|----------|
| 不知道第一句写什么 | 开头压力太大 | 先写第二段，最后补开头 |
| 写着写着跑题了 | 缺乏锚点 | 随时对照大纲，检查是否偏离 |
| 不知道怎么过渡 | 逻辑不清 | 用"连接词+观点"的结构 |
| 写得太慢 | 追求完美 | 先完成初稿，再修改润色 |
| 感觉写得很干 | 缺乏案例 | 暂停写作，先找2-3个案例 |

### 写作状态调整

#### 进入状态的方法

1. **仪式感**：固定时间、固定位置
2. **热身**：写 100 字随便什么内容
3. **目标明确**：告诉自己"今天只写 1000 字"

#### 卡住时的处理

1. **换一段写**：跳过卡住的地方
2. **大声朗读**：听听是否通顺
3. **休息一下**：散步5分钟再回来
4. **降低标准**：允许自己写垃圾

## 文章类型与写作模板

### 类型一：工具对比文

**适用场景**：比较两个或多个AI工具、产品的优劣。

**读者价值**：帮助读者做出选择决策。

**大纲模板**：

```
【标题】用了XX和XX后，我发现了X个惊人差异

【开头（150字以内）】
- 场景代入：什么场景下用的
- 冲突：有什么问题或发现
- 预告：本文会告诉你什么

【正文结构】
一、先说结论（300字）
- 2个工具的定位
- 我的推荐（给不同人群）

二、价格对比（500字）
- 官方价格
- 实际使用成本
- 哪个更划算

三、使用体验对比（800字）
- 速度对比
- 质量对比
- 稳定性对比
- 每个点用实际案例说明

四、适用场景（500字）
- 什么情况用A
- 什么情况用B
- 什么情况都用

五、最终建议（300字）
- 明确推荐
- 给出使用建议

【结尾】
- 总结
- 引导互动
```

**字数参考**：2000-3000字

### 类型二：情绪共鸣文

**适用场景**：讲述个人经历，引发读者情感共鸣。

**读者价值**：获得情感支持、行动启发。

**大纲模板**：

```
【标题】42岁被裁那天，我在XX坐了X小时

【开头（200字以内）】
- 场景：具体的时间、地点、动作
- 情绪：那一刻的感觉
- 转折：我决定...

【正文结构】
一、那个瞬间（400字）
- 详细描述发生了什么
- 细节越多越好（让人代入）
- 不是抱怨，是真实

二、为什么难（500字）
- 不是钱的问题，是...
- 不是丢脸，是...
- 点出深层痛点

三、我做了什么（600字）
- 经历了哪些挣扎
- 如何走出来的
- 关键转折点

四、给同样处境的你（500字）
- 不是鸡汤，是建议
- 具体可操作
- 真实不夸大

【结尾】
- 一句话总结
- 引导互动
```

**字数参考**：1800-2500字

### 类型三：热点解读文

**适用场景**：解读当前热点事件，提供独特视角。

**读者价值**：了解事件背后的逻辑、获得独特观点。

**大纲模板**：

```
【标题】XX刷屏了，我来泼点冷水/说点真话

【开头（150字以内）】
- 简单交代发生了什么
- 你的独特立场

【正文结构】
一、发生了什么（300字）
- 客观描述事件
- 关键时间线

二、为什么刷屏（400字）
- 分析传播原因
- 站在不同角度看

三、我的专业解读（800字）
- 你的独特角度
- 技术/商业/社会层面分析
- 不是复述新闻，是分析

四、对普通人的影响（500字）
- 利好哪些人
- 利空哪些人
- 需要注意什么

五、我的判断（300字）
- 预测未来
- 给出建议

【结尾】
- 观点总结
- 引导讨论
```

**字数参考**：2000-3000字

### 类型四：日常感悟文

**适用场景**：分享日常生活感悟、心得。

**读者价值**：获得启发、情感共鸣。

**大纲模板**：


## 常见问题

### 问题一：不知道写什么怎么办？

**答案**：从生活中找素材。从你的真实经历、遇到的问题、学到的东西入手。真实的故事最有共鸣。

### 问题二：追热点来不及写怎么办？

**答案**：如果超过48小时，就不要追了。可以换一个角度，写深度分析或后续跟进。

### 问题三：阅读量一直很低怎么办？

**答案**：检查标题是否有吸引力，检查开头是否抓住人，检查内容是否有干货。也可以参考同类高阅读量文章。

### 问题四：写不出来怎么办？

**答案**：先写大纲，大纲是骨架，先把骨架搭起来，再填肉。

### 问题五：数据焦虑怎么办？

**答案**：发完48小时后再看，前24小时不要频繁刷新后台。把精力放在下一篇文章上。

## 素材收集方法

### 信息源渠道

- **身边事**：自己的经历、朋友的故事、工作中遇到的问题
- **社交媒体**：评论区高频问题、热门讨论、话题标签
- **行业资讯**：36氪、虎嗅、InfoQ、少数派等
- **竞品分析**：同类爆款文章评论区、用户反馈

### 素材整理方法

1. 发现灵感 → 记录在手机备忘录（30秒内完成核心关键词）
2. 当天整理 → 扩展成 200 字素材卡片（时间、地点、人物、事件）
3. 写作时调用 → 素材库搜索关键词，选取相关素材

### 素材使用原则

- 优先用自己的真实经历，真实最有说服力
- 他人案例要注明来源或转述
- 数据要核实准确性，来源可靠
- 素材不在多，在于精准匹配选题

## 写作过程指南

### 开头写法

| 类型 | 模板 | 适用场景 |
|------|------|----------|
| 故事型 | "当时我正在...没想到..." | 情绪共鸣文 |
| 问题型 | "你有没有发现..." | 干货分享文 |
| 数据型 | "最新数据显示...意味着..." | 热点解读文 |
| 场景型 | "想象一下：周一早上..." | 日常感悟文 |

### 正文展开原则

1. **观点 + 案例循环**
   - 抛出一个核心观点（50字以内）
   - 跟一个真实案例佐证（100-200字）
   - 一句话总结这个观点（20-30字）

2. **段落节奏控制**
   - 重点段落：200-300字（充分论述）
   - 过渡段落：50-100字（承上启下）
   - 每段不超过 4 行（手机显示友好）

3. **常用连接词**
   - 递进：更重要的是、而且、事实上
   - 转折：但是、然而、不过
   - 因果：所以、正因如此、导致
   - 举例：比如、以...为例、就像

### 结尾模板

| 类型 | 示例 | 字数 |
|------|------|------|
| 总结型 | "回头看，核心就是..." | 50-80字 |
| 提问型 | "你觉得...？欢迎评论区..." | 30-50字 |
| 金句型 | "...这可能就是..." | 30-50字 |
| 号召型 | "如果觉得有用，欢迎..." | 30-50字 |

### 写作自查清单

完成初稿后快速检查：

- [ ] 开头 100 字内有钩子，能抓住人
- [ ] 每段有明确主题，段落长度合适
- [ ] 案例都是真实的，有细节支撑
- [ ] 没有正确的废话，每句话都有信息量
- [ ] 结尾有总结或引导互动

## 动态选题流程

当用户请求选题时，助手会自动执行以下流程：

1. **搜索阶段**：根据用户关注的领域（可选），搜索当前热门话题
2. **分析阶段**：评估每个话题的热度、时效性、竞争程度
3. **推荐阶段**：生成 3-5 个备选选题，带热度评分
4. **选择阶段**：用户选择或调整选题
5. **验证阶段**：执行选题决策 5 问流程
